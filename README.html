Core Features

A robust and scalable framework designed with industry best practices for reliable and maintainable automated tests.

Page Object Model (POM)

Separates UI interaction logic from test logic for better code organization and maintainability.

Data-Driven Testing

Reads test data from Excel files, allowing for easy expansion of test cases without code changes.

Configuration Management

Manages environment details like URLs and browser types via an external properties file.

Detailed HTML Reporting

Generates rich Extent Reports with test steps, data, and screenshots on failure.

Cross-Browser Support

Easily configurable to run tests on different browsers like Chrome, Firefox, and in headless mode.

Maven Integration

Uses Maven for streamlined dependency management and build lifecycle control.

Execution Workflow

Follow the step-by-step process from test initiation to report generation.

1. Execution Start

A user or CI/CD pipeline runs the `mvn clean test` command.

2. TestNG Suite

Maven invokes TestNG, which reads `testng.xml` to identify the test suite and attach listeners.

3. Browser Setup

`BaseTest` reads `config.properties`, initializes the correct WebDriver, and navigates to the URL.

4. Data Provision

`@DataProvider` uses `ExcelUtil` to read data from `testdata.xlsx` and feeds it to the test method.

5. Page Interaction

The test uses Page Object methods to perform actions. Each step is logged to the Extent Report.

6. Reporting & Teardown

The listener logs the final status. On failure, a screenshot is taken. Finally, the browser is closed.

Setup and Execution Guide

Prerequisites

  • Java Development Kit (JDK) 8 or higher
  • Apache Maven

Configuration

Open `src/main/resources/config.properties` to set the `browser` (`chrome`, `chrome-headless`, `firefox`) and `url` for testing.

Running Tests

Navigate to the project's root directory in your terminal and execute the following command:

mvn clean test
